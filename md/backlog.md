# Backlog

**Сборка:** вся сборка, миграции и тесты — только через Docker (`docker compose build`, `docker compose run --rm app alembic upgrade head`, `docker compose up -d` и т.д.). Не запускать pip/npm/alembic напрямую на хосте.

**Правило:** если предлагаешь запустить скрипт или команду — запускай сам, не перекладывай на пользователя.

**Правило:** когда таска выполнена — удалить её из списка ниже.

**Git:** при изменениях — атомарные коммиты (одна логическая правка = один коммит), осмысленные сообщения в стиле Conventional Commits (`feat:`, `fix:`, `refactor:`, `docs:`, `chore:`). Для крупных фич — ветка `feature/название` или `fix/название`, затем merge в `main`. После значимых изменений — сам делать `git add`, `git commit`, при необходимости `git push`.

## Рефакторинг Operations & Admin cabinet

**Цель:** чётко разделить «админ-кабинет» (просмотр и управление пользователями/драйверами/событиями) и «операции» (создание/ввод данных: event, participation, incident). Админ должен **отслеживать**: пользователи, драйверы, события, участия, инциденты, метрики; **менять**: профили пользователей, создавать события/участия/инциденты, при необходимости — лицензии, реклассификацию.
