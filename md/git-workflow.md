# Git workflow (RacerPath)

## Правила работы с репозиторием

1. **Атомарные коммиты** — один коммит = одна логическая правка (фича, багфикс, рефакторинг). Не смешивать в одном коммите несвязанные изменения.

2. **Сообщения коммитов (Conventional Commits):**
   - `feat: описание` — новая возможность
   - `fix: описание` — исправление бага
   - `refactor: описание` — рефакторинг без смены поведения
   - `docs: описание` — только документация
   - `chore: описание` — сборка, зависимости, конфиг
   - `test: описание` — тесты

3. **Ветки:**
   - `main` — стабильная ветка для продакшена.
   - Для крупной фичи или рискованного фикса: создать ветку `feature/название` или `fix/название`, коммитить туда, затем merge в `main`.

4. **После изменений** — самому выполнять `git add`, `git commit` с осмысленным сообщением. При необходимости — `git push` (если есть доступ).

5. **Не коммитить:** секреты (`.env`), артефакты сборки (`node_modules/`, `frontend/dist/`, бэкапы `_backup*/`). Они уже в `.gitignore`.
